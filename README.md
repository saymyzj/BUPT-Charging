# BUPT-Charging

北京邮电大学软件工程课程项目：智能充电桩调度计费系统。

本项目面向校园充电场景，围绕“缩短等待时间、提高充电桩利用率、完成透明计费”展开设计与实现，采用：

- 前端：Vue
- 后端：Python + Flask
- 数据库：SQLite
- 通信：HTTP + JSON

## 项目目标

- 支持快充/慢充两类请求
- 基于中央等待池和动态调度机制完成排队与分配
- 支持叫号确认、充电监控、详单账单、异常处理
- 支持联合验收场景下的单次模拟与批量模拟

```text
6. docs/联合验收统一接口与数据结构规范说明V2.md

- 服务端负责人
- 联合验收负责人

我的工作边界是：
- 主要负责 Flask 后端、SQLite 数据层、统一接口、批量模拟、账单详单生成、联合验收服务端落地
- 不主导调度算法规则本身
- 不主导前端页面实现
- 不负责整体对外沟通口径

请你输出一份非常详细的工作指导，必须包含以下内容：

1. 先根据现有文档，完整解释“成员B到底负责什么，不负责什么”
2. 梳理成员B需要接手的模块、接口、数据结构、文档位置
3. 给出成员B的“总体 TODO LIST”，要求按：
   - 立即做
   - 中期验收前做
   - 联合验收前做
   三个阶段拆开，并尽量细化
4. 告诉我如何正式启动开发：
   - 先看哪些文档
   - 先建哪些目录/文件
   - Flask 项目怎么起骨架
   - SQLite 怎么组织
   - 哪些接口应该最先写
5. 告诉我当务之急是什么，哪些事情现在绝对不能拖
6. 对每个核心接口给出服务端实现建议：
   - 输入校验
   - 状态流转
   - 可能的异常
   - 返回结构
7. 结合联合验收要求，详细说明：
   - 单次模拟怎么实现
   - 批量模拟怎么实现
   - 详单和账单怎么生成
   - summary 指标怎么计算
8. 结合 Mac / Windows 跨设备通信，说明作为服务端负责人现在该做哪些兼容准备
9. 结合 Git 规范，告诉我：
   - 我应该切哪个分支
   - 每天开发前后怎么操作
   - 什么时候合并到 develop
10. 请尽量给出按步骤执行的清单，而不是泛泛建议

输出要尽量详细，适合我直接照着执行。
```

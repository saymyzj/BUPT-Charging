## 二、用户端功能详细设计

### 2.1 F-U01 注册与登录

#### 功能描述

提供用户注册和登录能力，用于身份识别与历史数据关联。

#### 业务规则

1. 注册时需提供用户名和密码，用户名不可重复
2. 密码需存储为摘要形式（如 bcrypt），不存储明文
3. 登录成功后返回身份凭证（如 JWT Token），后续请求携带
4. 用户角色分为 `USER`（普通用户）和 `ADMIN`（管理员）
5. 一个用户同一时间只能有一个活跃的充电请求

#### 输入输出

**注册**

- 输入：`username`, `password`
- 输出：`user_id`, `username`, `role`, `created_at`

**登录**

- 输入：`username`, `password`
- 输出：`token`, `user_id`, `role`

#### 异常处理

- 用户名已存在：返回错误提示
- 密码格式不合规：返回格式要求
- 登录失败：返回用户名或密码错误

### 2.2 F-U02 提交充电请求

#### 功能描述

用户提交充电类型和申请充电量，系统生成充电请求并加入中央等待池。

#### 业务规则

1. 用户提交 `charge_mode`（`FAST` 或 `SLOW`）和 `request_energy`（单位 kWh）
2. 系统生成 `charge_request` 记录，初始状态为 `PENDING`
3. 请求按 `charge_mode` 加入对应中央等待池（`FAST_POOL` 或 `SLOW_POOL`），状态变为 `WAITING`
4. 调度器触发一次预测排程，计算该请求的预计等待时间、预计开始时间和预计完成时间
5. `request_energy` 仅作为调度预测依据，不代表最终实际充电量
6. 同一用户不可重复提交未完成的请求

#### 输入输出

遵循当前冻结接口规范：

- 接口：`POST /api/request/create`
- 输入：`request_time`, `charge_mode`, `request_energy`
- 输出：`request_id`, `status`, `estimated_wait_seconds`, `estimated_start_time`, `estimated_finish_time`

#### 约束

- `charge_mode` 仅接受 `FAST` 或 `SLOW`
- `request_energy` 必须为正数，建议上限为 100 kWh
- 系统为请求自动分配 `battery_limit_energy`（模拟车辆可充入上限）

3. 显示每种类型当前等待池中的排队人数

| 展示项 | 说明 |

| 桩类型 | 快充 / 慢充 |

### 2.4 F-U04 查看预计等待时间

#### 功能描述

用户在排队期间可实时查看当前预计等待时长，系统在每次重调度后更新该值。

#### 业务规则

1. 预计等待时间由调度器在预测排程阶段计算
2. 计算公式：`pred_wait(u, s) = max(0, available_time(s) - now)`
3. 系统对同类型所有可用桩进行模拟分配，取最优桩的预测结果
4. 每次重调度事件发生后，等待时间自动更新
5. 前端应定期轮询或通过通知机制获取最新值

#### 输入输出

通过查询请求状态接口获取：

- 接口：`GET /api/request/status/{request_id}`
- 输出中包含：`estimated_wait_seconds`

---

### 2.5 F-U05 查看排队状态

#### 功能描述

展示用户当前请求的完整状态信息。

#### 业务规则

1. 状态遵循统一请求状态枚举：`PENDING` → `WAITING` → `CALLED` → `CONFIRMED` → `CHARGING` → `COMPLETED` / `COMPLETED_EARLY`
2. 异常分支状态：`CANCELLED`, `NO_SHOW`, `INTERRUPTED`, `FAULT_REQUEUE`
3. 展示当前状态、预计等待时间、预计开始时间、预计完成时间
4. 若已分配桩号，展示桩编号
5. 若已被叫号，展示叫号时间和确认倒计时

#### 展示内容

| 展示项 | 说明 |
|--------|------|
| 请求状态 | 当前状态文本和图标 |
| 预计等待时间 | 倒计时或分钟数 |
| 预计开始时间 | 时间戳 |
| 预计完成时间 | 时间戳 |
| 分配桩号 | 如已分配则显示 |
| 排队位置 | 在等待池中的相对位置 |

### 2.6 F-U06 临近确认（叫号确认）

#### 功能描述

当某个充电桩即将可用时，系统向当前最合适的用户发起叫号通知，用户需在限定时间内确认到场。

#### 业务规则

1. 触发条件：某桩距可用时刻小于阈值 `T_call`（可配置，默认 5 分钟）
2. 系统从等待池中选择优先级最高的用户，发送叫号通知
3. 请求状态变为 `CALLED`
4. 用户需在 `confirm_timeout`（可配置，默认 3 分钟）内确认
5. 确认后状态变为 `CONFIRMED`，系统为该桩临时保留 `arrival_timeout`（可配置，默认 5 分钟）
6. 用户在保留时间内完成接入，开始充电
7. 若未确认或超时未到场，判为过号（`NO_SHOW`），系统选择下一个用户

#### 输入输出

遵循当前冻结接口规范：

- 接口：`POST /api/request/confirm_arrival`
- 输入：`request_id`, `confirm_time`
- 输出：`request_id`, `status`, `station_id`

#### 时间参数

| 参数 | 含义 | 默认值 |
|------|------|--------|
| `T_call` | 提前叫号阈值 | 5 分钟 |
| `confirm_timeout` | 确认超时时间 | 3 分钟 |
| `arrival_timeout` | 到场超时时间 | 5 分钟 |

### 2.7 F-U07 开始充电与充电监控

#### 功能描述

用户到场接入后开始充电，系统创建充电会话并实时记录充电进度。

#### 业务规则

1. 用户确认到场后，系统创建 `charging_session` 记录
2. 请求状态变为 `CHARGING`，桩状态变为 `CHARGING`
3. 充电过程中实时更新 `actual_energy`（已充电量）
4. 预计剩余时间 = `(request_energy - actual_energy) / power(s)`
5. 若 `actual_energy` 达到 `battery_limit_energy`，提前结束充电
6. 充电完成后状态变为 `COMPLETED`（正常完成）或 `COMPLETED_EARLY`（提前充满）
7. 会话结束后桩状态变为 `WAITING_TO_LEAVE`

#### 实时监控展示

| 展示项 | 说明 |
|--------|------|
| 当前充电量 | `actual_energy` kWh |
| 目标充电量 | `request_energy` kWh |
| 充电进度 | 百分比 |
| 已充电时长 | 从开始充电到当前的时长 |
| 预计剩余时长 | 按当前功率估算 |
| 实时费用 | 按已充电量和计费规则估算 |
| 充电桩编号 | 当前使用的桩号 |
| 充电功率 | 当前桩的额定功率 |

---

### 2.8 F-U08 取消排队

#### 功能描述

用户在尚未开始充电时可取消排队。

#### 业务规则

1. 仅在 `WAITING` 或 `CALLED` 状态下允许取消
2. 若已处于 `CHARGING` 状态，应使用中断充电功能
3. 取消后请求状态更新为 `CANCELLED`
4. 从等待池中移除该请求
5. 若已被分配桩号（`CALLED` 或 `CONFIRMED` 状态），释放桩预留
6. 触发重调度，更新其他用户的预计等待时间

#### 输入输出

遵循当前冻结接口规范：

- 接口：`POST /api/request/cancel_queue`
- 输入：`request_id`, `cancel_time`
- 输出：`request_id`, `status`

---

### 2.9 F-U09 中断充电

#### 功能描述

用户在充电过程中可主动结束充电。

#### 业务规则

1. 仅在 `CHARGING` 状态下允许中断
2. 中断后按已充电量和实际时长结算费用
3. 请求状态变为 `INTERRUPTED`
4. 充电桩释放，状态变为 `IDLE`（或 `WAITING_TO_LEAVE` 如需挪车）
5. 系统生成会话记录，`interrupt_reason` 记为 `USER_INTERRUPT`
6. 触发重调度

#### 输入输出

遵循当前冻结接口规范：

- 接口：`POST /api/request/interrupt_charge`
- 输入：`request_id`, `interrupt_time`
- 输出：`request_id`, `status`, `actual_energy`, `actual_service_seconds`

### 2.14 F-U14 申报量与实际量差异处理

#### 功能描述

处理用户申报充电量与实际充电量不一致的场景。该场景为系统常规场景，非异常输入。

#### 业务规则

1. `request_energy` 用于调度预测阶段的排队和等待时间估算
2. 充电过程中实时记录 `actual_energy`
3. 系统维护 `battery_limit_energy` 模拟车辆可充入上限
4. 实际充电量 = `min(request_energy, battery_limit_energy)`
5. 若车辆提前充满（`actual_energy` 达到 `battery_limit_energy`）：
   - 会话提前结束，状态为 `COMPLETED_EARLY`
   - 充电桩提前释放
   - 触发重调度，更新后续用户预计等待时间
6. 最终费用按 `actual_energy` 和实际充电时长结算

## 三、管理端功能详细设计

### 3.2 F-A02 调度参数配置

#### 功能描述

管理员可配置调度算法参数和验收场景参数。

#### 可配置参数

| 参数 | 含义 | 默认值 |
|------|------|--------|

### 3.3 F-A03 计费规则管理

#### 功能描述

管理员配置计费模式和费率。

#### 计费模式

| 模式 | 说明 |
|------|------|
| `ENERGY` | 按电量计费，单位 元/kWh |
| `TIME` | 按时长计费，单位 元/分钟 |
| `MIXED` | 电量费 + 时长费 |

#### 可配置参数

| 参数 | 含义 | 默认值 |
|------|------|--------|
| `billing_mode` | 计费模式 | `ENERGY` |
| `energy_price` | 电量单价（元/kWh） | 1.5 |
| `time_price` | 时长单价（元/分钟） | 0.0 |
| `occupancy_price` | 占位费单价（元/分钟） | 0.5 |
| `occupancy_start_after` | 超时多久开始计占位费（分钟） | 10 |

#### 计费公式

- 电量费用：`energy_fee = actual_energy * energy_price`
- 时长费用：`time_fee = actual_service_minutes * time_price`
- 占位费用：`occupancy_fee = max(0, overtime_minutes) * occupancy_price`
- 总费用：`total_fee = energy_fee + time_fee + occupancy_fee`

### 3.4 F-A04 异常事件处理

#### 功能描述

管理员查看并处理系统中的异常事件。

#### 异常事件类型

| 事件类型 | 说明 | 处理方式 |
|----------|------|----------|
| 用户过号 | 叫号后未确认或未到场 | 自动重选下一用户 |
| 用户取消 | 排队中取消请求 | 从等待池移除，触发重调度 |
| 用户中断 | 充电中主动结束 | 按实际量结算，释放桩 |
| 超时未挪车 | 充电完成后未及时挪走 | 计入占位费，桩不可重新分配 |
| 充电桩故障 | 桩发生故障 | 中断当前会话，用户重入队获补偿优先级 |
| 故障恢复 | 桩故障修复 | 恢复 `IDLE`，重新参与调度 |

#### 充电桩故障处理详细规则

1. 若故障发生在空闲态：直接标记 `FAULT`，不参与调度
2. 若故障发生在充电中：
   - 记录已充电量 `actual_energy`
   - 计算剩余电量 `remaining = request_energy - actual_energy`
   - 将剩余电量生成新请求重新入池，状态为 `FAULT_REQUEUE`
   - 新请求增加 `fault_bonus` 优先级补偿
3. 故障恢复后：桩状态恢复为 `IDLE`，触发重调度

## 五、核心业务流程设计

### 5.1 正常充电完整流程

```
用户提交请求
  → 生成 charge_request (PENDING)
  → 加入等待池 (WAITING)
  → 调度器预测排程
  → 用户查看预计等待时间
  → 桩即将可用，系统叫号 (CALLED)
  → 用户确认到场 (CONFIRMED)
  → 桩可用，开始充电 (CHARGING)
  → 充电过程实时监控
  → 充电完成 (COMPLETED / COMPLETED_EARLY)
  → 提醒挪车 (WAITING_TO_LEAVE)
  → 用户确认挪车
  → 桩释放 (IDLE)
  → 生成详单和账单
  → 用户支付 (PAID)
```

### 5.2 异常分支流程

#### 5.2.1 用户取消排队

```
WAITING / CALLED → 用户取消 → CANCELLED
  → 从等待池移除
  → 释放桩预留（如有）
  → 触发重调度
```

#### 5.2.2 用户过号

```
CALLED → 确认超时 → NO_SHOW
  → 可重新入池 (WAITING)，附加轻度惩罚
  → 或完全退出
  → 系统选择下一用户叫号
```

#### 5.2.3 用户中断充电

```
CHARGING → 用户主动中断 → INTERRUPTED
  → 按已充电量结算
  → 释放充电桩
  → 触发重调度
```

#### 5.2.4 提前充满

```
CHARGING → actual_energy >= battery_limit → COMPLETED_EARLY
  → 提前结束会话
  → 充电桩提前释放
  → 触发重调度
  → 更新后续用户预计等待时间
```

#### 5.2.5 超时未挪车

```
COMPLETED → 进入 WAITING_TO_LEAVE → 超过 leave_buffer
  → 开始计入占位费
  → 桩状态保持 WAITING_TO_LEAVE，不可重新分配
  → 用户确认挪车后，桩恢复 IDLE
```

#### 5.2.6 充电桩故障

```
CHARGING → 桩故障 → 会话中断
  → 记录已充电量
  → 剩余电量生成新请求，状态 FAULT_REQUEUE
  → 新请求获 fault_bonus 补偿
  → 桩标记 FAULT
  → 触发重调度

故障恢复 → 桩恢复 IDLE → 触发重调度
```

### 5.3 事件驱动重调度

以下事件触发调度器重新计算：

| 序号 | 事件 | 调度动作 |
|------|------|----------|
| 1 | 新用户提交请求 | 更新等待池，重算预测 |
| 2 | 用户取消排队 | 移除请求，重算预测 |
| 3 | 用户确认叫号 | 锁定桩分配 |
| 4 | 用户过号 | 释放桩预留，重选用户 |
| 5 | 用户开始充电 | 更新桩状态和可用时间 |
| 6 | 用户中断充电 | 释放桩，重算预测 |
| 7 | 充电完成 | 更新桩可用时间，选择下一用户 |
| 8 | 用户超时未挪车 | 桩可用时间后移 |
| 9 | 充电桩故障 | 标记桩不可用，受影响用户重入队 |
| 10 | 故障恢复 | 桩恢复可用，重算预测 |
| 11 | 用户提前充满 | 桩提前释放，重算预测 |

## 六、状态机设计

### 6.1 请求状态机

```
PENDING → WAITING → CALLED → CONFIRMED → CHARGING → COMPLETED
                                                   → COMPLETED_EARLY
                  → CANCELLED (from WAITING or CALLED)
            CALLED → NO_SHOW → WAITING (重新入队)
                             → CANCELLED (放弃)
           CHARGING → INTERRUPTED
           CHARGING → FAULT_REQUEUE → WAITING (新请求)
```

| 状态 | 含义 | 可转换到 |
|------|------|----------|
| `PENDING` | 已提交，尚未入池 | `WAITING` |
| `WAITING` | 在等待池中 | `CALLED`, `CANCELLED` |
| `CALLED` | 已被叫号 | `CONFIRMED`, `NO_SHOW`, `CANCELLED` |
| `CONFIRMED` | 已确认到场 | `CHARGING` |
| `CHARGING` | 正在充电 | `COMPLETED`, `COMPLETED_EARLY`, `INTERRUPTED`, `FAULT_REQUEUE` |
| `COMPLETED` | 正常完成 | 终态 |
| `COMPLETED_EARLY` | 提前充满 | 终态 |
| `CANCELLED` | 用户取消 | 终态 |
| `NO_SHOW` | 过号 | `WAITING`（重入队）或终态 |
| `INTERRUPTED` | 中断充电 | 终态 |
| `FAULT_REQUEUE` | 故障重入队 | `WAITING`（新请求） |

### 6.2 充电桩状态机

```
IDLE → RESERVED → CHARGING → WAITING_TO_LEAVE → IDLE
                           → IDLE (中断后直接释放)
任意状态 → FAULT → IDLE (故障恢复)
```

| 状态 | 含义 | 可转换到 |
|------|------|----------|
| `IDLE` | 空闲 | `RESERVED`, `FAULT` |
| `RESERVED` | 已预留等待接入 | `CHARGING`, `IDLE`（过号释放）, `FAULT` |
| `CHARGING` | 充电中 | `WAITING_TO_LEAVE`, `IDLE`（中断）, `FAULT` |
| `WAITING_TO_LEAVE` | 待挪车 | `IDLE`, `FAULT` |
| `FAULT` | 故障 | `IDLE`（恢复） |

### 6.3 支付状态

```
UNPAID → PAID
UNPAID → FAILED → UNPAID (重试)
```

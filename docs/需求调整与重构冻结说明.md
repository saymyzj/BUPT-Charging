# 智能充电桩调度计费系统：需求调整与重构冻结说明

> 状态：最终冻结  
> 版本：V3.0  
> 冻结日期：2026-04-19  
> 适用范围：后续代码重构、前后端联调、系统验收、部署上线

## 一、背景说明

项目已经进入开发过程中期，前后端已有一定实现、页面已有部分落地、组内协作与增量模型也已形成稳定节奏。  
但老师最终确认的要求与当前旧设计存在结构性差异，已不是“局部补丁”能够解决的问题，而是一次明确的**需求调整驱动的系统重构**。

本次重构只先冻结文档，不先动代码。  
后续所有代码重构、接口调整、页面联调、验收准备与部署上线，均必须以新 `docs/` 为唯一依据。

## 三、本轮重构冻结的核心结论

### 3.1 正式运行模型

系统正式采用：

- 等候区
- 充电区
- 每个充电桩一条固定长度为 `ChargingQueueLen` 的队列

其中：

- `ChargingQueueLen` 包含队首正在充电或可立即充电的位置
- 系统不再以“中央等待池 + 动态分配”作为正式运行模型

### 3.2 排队号与顺序

- 快充用户进入等候区后生成 `F` 号：`F1、F2、...`
- 慢充用户进入等候区后生成 `T` 号：`T1、T2、...`
- 同一模式排队号在系统运行期间单调递增，不复用
- 系统重置或测试场景重置后，从 `F1/T1` 重新开始编号
- 排队号是普通调度、故障重调度、用户展示、验收回放的唯一顺序依据

### 3.3 普通调度

普通模式下：

1. 只在对应充电模式内调度
2. 当某个同模式充电桩队列出现空位时，从该模式等候区取排队号最靠前的车辆
3. 将该车辆加入能使其“等待时间 + 自身充电时间”最短的同模式桩队列

### 3.4 故障调度

系统同时支持两种故障策略：

- `PRIORITY`
- `TIME_ORDER`

并由管理员运行时切换。

故障恢复时，不论当前策略为何，统一按时间顺序重调度。

### 3.5 扩展调度

系统正式冻结三种调度模式：

- `NORMAL`
- `EXT_SINGLE_BATCH`
- `EXT_FULL_BATCH`

其中：

- `EXT_SINGLE_BATCH` 对应“单次调度总充电时长最短”
- `EXT_FULL_BATCH` 对应“批量调度总充电时长最短”

两种扩展模式均为完整可实现模式，不再只作为口头说明保留。

### 3.6 计费

账单正式口径为：

- `charge_fee`
- `service_fee`
- `total_fee`

并严格按最终要求计算：

- 峰时：`1.0`
- 平时：`0.7`
- 谷时：`0.4`
- 服务费单价：`0.8`

### 3.7 部署上线

部署上线被正式纳入本轮冻结范围：

- 单台阿里云 ECS 一体化部署
- clouDNS 免费域名与 DNS
- 单域名同源
- 必须 HTTPS
- 前端由 `Nginx` 托管
- `/api` 由 `Nginx` 反代到后端 Flask 服务
- 上线继续使用 SQLite
- 发布方式固定为手动发布 + `systemd` + `Nginx`
- 当前方案不引入 OSS

### 3.8 V3.0 新增决策摘要（D1—D12）

本轮从 V2.0 升级到 V3.0 时，经组内评审确认了以下 12 项决策：

| 编号 | 决策项 | 结论 |
|------|--------|------|
| D1 | 功能编号体系 | 在功能设计.md 建立完整 `F-U / F-A / F-J` 编号表，所有文档统一引用 |
| D2 | 充电区"取消"语义 | 等价于"提前结束并结算"，状态 `COMPLETED_EARLY`，共用 `/api/request/stop` |
| D3 | 用户/车辆信息维护 | 管理员端新增 F-A07，含列表查看与车辆电池容量维护 |
| D4 | 系统参数入口 | 启动配置文件加载 + 管理员只读查看；运行时不可改 |
| D5 | 状态机图 | 在功能设计.md 新增状态机专项章节（请求状态 + 桩状态） |
| D6 | 详单/账单字段 | 在冻结接口文档.md 新增细粒度字段字典（含义/类型/单位/特殊场景值） |
| D7 | 预计时间展示 | `estimated_wait/start/finish` 正式列为用户端可见字段 |
| D8 | openapi.yaml 定位 | 机器可读附录，Markdown 为主，冲突时以 Markdown 为准 |
| D9 | 报表粒度 | 严格只做 `day / week / month` |
| D10 | 请求状态枚举 | 保持 7 个不变 |
| D11 | 版本号 | 全部统一到 V3.0、冻结日期 2026-04-19 |
| D12 | 文档清单变动 | 新增部署上线手册.md、故障与扩展场景指南.md；更新冻结说明到 V3.0 |

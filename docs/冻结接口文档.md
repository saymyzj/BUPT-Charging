## 二、统一约定

### 2.1 通信方式

- 协议：HTTPS
- 数据格式：JSON
- 字符集：UTF-8
- 域名形态：单域名同源
- 前端资源：`Nginx` 直接托管
- 后端接口：`/api` 反向代理到 Flask 服务

### 2.2 响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 2.3 时间与单位

- 时间格式：ISO 8601，如 `2026-04-19T10:00:00`
- 电量单位：度 / kWh
- 功率单位：度每小时 / kW
- 时长单位：秒
- 金额单位：元

## 三、统一枚举

### 3.1 充电模式 `charge_mode`

- `FAST`
- `SLOW`

### 3.2 调度模式 `dispatch_mode`

- `NORMAL`
- `EXT_SINGLE_BATCH`
- `EXT_FULL_BATCH`

### 3.3 故障调度策略 `fault_dispatch_mode`

- `PRIORITY`
- `TIME_ORDER`

### 3.4 请求状态 `request_status`

- `WAITING_AREA`：在等候区等待
- `QUEUED`：已进入某充电桩队列但未开始充电
- `CHARGING`：正在充电
- `COMPLETED`：正常完成
- `COMPLETED_EARLY`：提前结束或提前充满后完成结算（包含充电区取消）
- `CANCELLED`：在等候区取消
- `FAULT_INTERRUPTED`：因充电桩故障中断，本次详单已结束

### 3.5 充电桩状态 `station_status`

- `RUNNING`
- `SHUTDOWN`
- `FAULT`

### 3.6 用户角色 `role`

- `USER`
- `ADMIN`

## 四、统一错误码

| code | 含义 |
|------|------|
| 0 | 成功 |
| 1001 | 参数错误（格式/字段缺失） |
| 1002 | 请求不存在 |
| 1003 | 当前状态不允许该操作 |
| 1004 | 等候区容量已满 |
| 1005 | 当前模式无可用充电桩 |
| 1006 | 充电桩故障 |
| 1007 | 充电桩未处于可关闭状态 |
| 1008 | `request_energy` 不合法（≤0 或超出允许区间） |
| 1009 | 系统参数仅支持启动时注入，运行时不可改 |
| 1010 | 用户有活跃请求，不可修改其车辆电池容量 |
| 1099 | 服务器内部错误 |

## 五、用户端接口

### 5.1 注册

`POST /api/auth/register`

请求：

```json
{
  "username": "user01",
  "password": "123456",
  "battery_capacity": 60.0
}
```

响应 `data`：

- `user_id`
- `username`
- `role`
- `created_at`

### 5.2 登录

`POST /api/auth/login`

请求：

```json
{
  "username": "user01",
  "password": "123456"
}
```

响应 `data`：

- `token`
- `user_id`
- `role`

### 5.3 创建充电请求

`POST /api/request/create`

请求：

```json
{
  "request_time": "2026-04-19T10:00:00",
  "charge_mode": "FAST",
  "request_energy": 20.0
}
```

校验：`request_energy ∈ (0, 300]` kWh，否则返回 1008。

响应 `data`：

- `request_id`
- `queue_number`
- `request_status`
- `front_waiting_count`

### 5.4 查询请求状态

`GET /api/request/status/{request_id}`

响应 `data` 字段：

- `request_id`
- `queue_number`
- `charge_mode`
- `request_energy`
- `request_status`
- `front_waiting_count`
- `station_code`
- `station_queue_position`
- `estimated_wait_seconds`
- `estimated_start_time`
- `estimated_finish_time`

### 5.5 等候区修改充电模式

`PUT /api/request/mode`

请求：

```json
{
  "request_id": "REQ0001",
  "charge_mode": "SLOW"
}
```

规则：

- 仅允许 `WAITING_AREA`
- 修改后重新生成 `F/T` 排队号，并排到新模式队尾

### 5.6 等候区修改请求电量

`PUT /api/request/energy`

请求：

```json
{
  "request_id": "REQ0001",
  "request_energy": 25.0
}
```

规则：

- 仅允许 `WAITING_AREA`
- 保留原排队号不变
- 同样校验 1008

### 5.7 等候区取消请求

`POST /api/request/cancel`

规则：

- 仅允许 `WAITING_AREA`
- 成功后状态变为 `CANCELLED`
- 不生成详单

### 5.8 充电区取消 / 提前结束

`POST /api/request/stop`

规则：

- 允许状态：`CHARGING`、`QUEUED`
- 按已充电量结算，生成独立详单
- 状态进入 `COMPLETED_EARLY`
- "充电区取消"与"充电区提前结束"语义等价，共用本接口

请求体可选 `intent` 字段（仅作日志标注，不影响结算）：

```json
{
  "request_id": "REQ0001",
  "intent": "USER_CANCEL"
}
```

`intent` 可选值：`USER_STOP`（自然提前结束）、`USER_CANCEL`（充电区取消）。

### 5.9 查询详单

`GET /api/request/detail/{request_id}`

响应 `data` 字段参见 §9 详单字段字典。

## 九、详单字段字典

| 字段 | 类型 | 单位 | 含义与来源 | 特殊场景写法 |
|------|------|------|-----------|-------------|
| `detail_id` | string | — | 详单编号，系统生成，全局唯一 | 所有场景都必填 |
| `detail_generated_at` | string(ISO8601) | — | 详单生成时刻，对应计费完成瞬间 | 所有场景都必填 |
| `station_code` | string | — | 实际使用的充电桩编号 | 提前结束/故障中断都写实际承载桩 |
| `actual_energy` | number | kWh | 实际充电度数 | 正常完成=`request_energy`；提前结束=已充累计；故障中断=到故障发生时刻的累计 |
| `charge_duration_seconds` | integer | s | 从 `start_time` 到 `stop_time` 的有效充电秒数 | 故障中断=从开始到故障时刻 |
| `start_time` | string(ISO8601) | — | 本次充电开始时刻 | 所有场景都是车辆真正开始充电的时刻 |
| `stop_time` | string(ISO8601) | — | 本次充电结束时刻 | 正常完成=充满时刻；提前结束/充电区取消=停止瞬间；故障中断=故障瞬间 |
| `charge_fee` | number | 元 | 充电费，按峰/平/谷分段求和 | 跨时段线性切分后求和 |
| `service_fee` | number | 元 | 服务费，`0.8 × actual_energy` | 所有场景同公式 |
| `total_fee` | number | 元 | `charge_fee + service_fee` | 所有场景同公式 |
| `request_status` | string | — | 本条详单对应的最终请求状态 | `COMPLETED` / `COMPLETED_EARLY` / `FAULT_INTERRUPTED` 三者之一；`CANCELLED` 不生成详单 |

> 说明：`detail_id` 与 `request_id` 不是 1:1 关系。故障中断会产生一条 `FAULT_INTERRUPTED` 详单后，故障重调度后的续充是一条新请求、对应一条新的 `COMPLETED` 或 `COMPLETED_EARLY` 详单。

## 十、账单字段字典

账单即详单的费用切片，本系统不单独建表。下列字段全部来源于详单字段：

| 字段 | 公式 | 说明 |
|------|------|------|
| `charge_fee` | `Σ(unit_price_slot × energy_in_slot)` | 按充电时段落在峰/平/谷区间求和 |
| `service_fee` | `0.8 × actual_energy` | 单价 0.8 元/度，覆盖实际充电度数 |
| `total_fee` | `charge_fee + service_fee` | 最终应收 |

跨时段拆分规则：

1. 记录 `start_time` 与 `stop_time`
2. 将 `[start_time, stop_time]` 按照峰/平/谷的固定时段边界切成若干子区间
3. 每个子区间内的秒数 × 充电功率 ÷ 3600 = 该子区间实际充电度数
4. 子区间度数 × 对应单价 = 该子区间充电费
5. 子区间充电费求和 = `charge_fee`

此规则对"跨天（谷时跨 `00:00`）"同样适用。

## 十一、报表字段字典

| 字段 | 类型 | 说明 |
|------|------|------|
| `time_key` | string | 粒度键：`day=YYYY-MM-DD`、`week=YYYY-Www`、`month=YYYY-MM` |
| `station_code` | string | 充电桩编号 |
| `total_charge_count` | integer | 本粒度内结算的详单数（含 `COMPLETED` / `COMPLETED_EARLY` / `FAULT_INTERRUPTED`） |
| `total_charge_seconds` | integer | 本粒度内该桩的有效充电总秒数 |
| `total_charge_energy` | number | 本粒度内该桩的充电总度数 |
| `total_charge_fee` | number | 本粒度内该桩的充电费总和 |
| `total_service_fee` | number | 本粒度内该桩的服务费总和 |
| `total_fee` | number | `total_charge_fee + total_service_fee` |

粒度选择规则：

- `day`：按本地日历日聚合
- `week`：按 ISO 周（周一到周日）聚合，`time_key = YYYY-Www`
- `month`：按本地日历月聚合

## 十二、废弃口径说明

以下内容不再作为正式冻结字段或语义（已在 V3.0 彻底去除）：

- `FAST_POOL`、`SLOW_POOL`
- 中央等待池
- `energy_fee` / `time_fee` / `occupancy_fee`
- `CALLED` / `CONFIRMED` / `NO_SHOW` / `WAITING_TO_LEAVE` / `FAULT_REQUEUE`
- `battery_limit_energy`
- 模拟支付、虚拟余额

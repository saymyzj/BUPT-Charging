## 二、统一约定

### 2.1 通信方式

- 协议：HTTP
- 数据格式：JSON
- 字符集：UTF-8
- 服务端监听：`0.0.0.0:<port>`
- 统一响应结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 2.2 认证约定

- `/health`、批量测试接口和核心演示接口可先支持免认证模式，便于调试与验收
- 用户端和管理端扩展接口使用 JWT
- Header：

```http
Authorization: Bearer <token>
```

### 2.3 时间格式

统一采用 ISO 8601 字符串：

```text
2026-03-30T10:00:00
```

### 2.4 数值单位

| 字段类型 | 单位 |
|----------|------|
| 能量 | kWh |
| 功率 | kW |
| 时间长度 | 秒 |
| 费用 | 元 |

## 三、统一枚举表

### 3.1 充电模式 `charge_mode`

| 值 | 中文含义 |
|----|----------|
| `FAST` | 快充 |
| `SLOW` | 慢充 |

### 3.2 请求状态 `status`

| 值 | 中文含义 | 说明 |
|----|----------|------|
| `PENDING` | 已提交 | 已创建请求但尚未正式进入预测 |
| `WAITING` | 等待中 | 已进入等待池 |
| `CALLED` | 已叫号 | 服务端已发出叫号提醒 |
| `CONFIRMED` | 已确认到场 | 用户已确认到场，等待接入或已预留桩 |
| `CHARGING` | 充电中 | 已开始充电 |
| `COMPLETED` | 已完成 | 正常充电完成 |
| `COMPLETED_EARLY` | 提前完成 | 因提前充满而结束 |
| `CANCELLED` | 已取消 | 用户取消排队 |
| `NO_SHOW` | 过号 | 用户未及时确认或未及时到场 |
| `INTERRUPTED` | 已中断 | 用户主动或被动中断充电 |
| `FAULT_REQUEUE` | 故障重排 | 因桩故障重新入队 |

### 3.3 充电桩状态 `station_status`

| 值 | 中文含义 | 说明 |
|----|----------|------|
| `IDLE` | 空闲 | 可立即接单 |
| `RESERVED` | 已预留 | 已为某请求暂时保留 |
| `CHARGING` | 充电中 | 当前桩正在使用 |
| `WAITING_TO_LEAVE` | 待挪车 | 用户充电已结束但尚未挪车 |
| `FAULT` | 故障 | 当前桩不可用 |

### 3.4 支付状态 `payment_status`

| 值 | 中文含义 |
|----|----------|
| `UNPAID` | 未支付 |
| `PAID` | 已支付 |
| `FAILED` | 支付失败 |

### 3.5 计费模式 `billing_mode`

| 值 | 中文含义 |
|----|----------|
| `ENERGY` | 按电量计费 |
| `TIME` | 按时长计费 |
| `MIXED` | 混合计费 |

## 四、统一错误码

| code | 含义 | 处理建议 |
|------|------|----------|
| 0 | 成功 | 正常处理 |
| 1001 | 请求参数错误 | 检查必填字段和字段类型 |
| 1002 | 请求不存在 | 检查 `request_id` 是否存在 |
| 1003 | 当前状态不允许该操作 | 例如对已完成请求执行取消 |
| 1004 | 过号，无法确认到场 | 前端提示重新发起或等待重新入队 |
| 1005 | 已取消，无法继续操作 | 前端停止继续流程 |
| 1006 | 充电桩故障 | 记录异常，触发重排或提示稍后再试 |
| 1007 | 支付失败 | 提示余额不足或稍后重试 |
| 1099 | 服务器内部错误 | 记录日志并提示重试 |

### 5.3 组内冻结约定：调度模块

为避免成员A和成员B在后端实现阶段出现理解分歧，以下组内约定视为冻结：

1. 创建请求时立即触发一次预测排程
2. 关键事件发生后统一触发一次重调度
3. 批量模拟按 `request_time` 逐事件回放，并在每个事件处理完成后重算预测结果
4. `request_time` 是业务时间，单次请求中不得被服务器当前时间随意覆盖
5. `waiting_area_capacity` 是系统全局等候区容量，由快充池和慢充池共享
6. `CALLED` 状态当前阶段不计入等待区容量占用
7. 调度模块最小输入输出约定见 [调度模块输入输出约定.md](/Users/zhoujia/code/SE/ChargingPile/docs/调度模块输入输出约定.md)

## 6.2 提交充电请求

### 请求

`POST /api/request/create`

```json
{
  "request_time": "2026-03-30T10:00:00",
  "charge_mode": "FAST",
  "request_energy": 20.0
}
```

### 请求字段说明

| 字段 | 类型 | 必填 | 含义 | 规则 |
|------|------|------|------|------|
| `request_time` | string | 是 | 发起请求时间 | ISO 8601 |
| `charge_mode` | string | 是 | 充电模式 | `FAST` / `SLOW` |
| `request_energy` | number | 是 | 申报充电量 | 必须大于 0 |

### 响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "request_id": "REQ001",
    "status": "WAITING",
    "estimated_wait_seconds": 600,
    "estimated_start_time": "2026-03-30T10:10:00",
    "estimated_finish_time": "2026-03-30T11:10:00"
  }
}
```

### 响应字段说明

| 字段 | 类型 | 含义 |
|------|------|------|
| `data.request_id` | string | 请求编号 |
| `data.status` | string | 当前请求状态 |
| `data.estimated_wait_seconds` | integer | 预计等待秒数 |
| `data.estimated_start_time` | string | 预计开始充电时间 |
| `data.estimated_finish_time` | string | 预计结束时间 |

### 可能情况与处理

| 情况 | 说明 | 处理 |
|------|------|------|
| `charge_mode` 非法 | 不是 `FAST`/`SLOW` | 返回 `1001` |
| `request_energy <= 0` | 非法电量 | 返回 `1001` |
| 用户已有未完成请求 | 重复请求 | 返回 `1003`，提示先完成当前请求 |
| 所有桩故障 | 无法预测正常排队 | 仍可进入等待态，但需在消息中说明资源不可用 |

## 6.3 查询请求状态

### 请求

`GET /api/request/status/{request_id}`

### 响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "request_id": "REQ001",
    "status": "CALLED",
    "station_id": "FAST_01",
    "estimated_wait_seconds": 0,
    "last_called_time": "2026-03-30T10:08:00"
  }
}
```

### 响应字段说明

| 字段 | 类型 | 含义 |
|------|------|------|
| `data.request_id` | string | 请求编号 |
| `data.status` | string | 当前状态 |
| `data.station_id` | string/null | 当前分配或预分配的充电桩编号 |
| `data.estimated_wait_seconds` | integer | 当前剩余预计等待秒数 |
| `data.last_called_time` | string/null | 最近一次叫号时间 |

### 可能情况与处理

| 情况 | 说明 | 处理 |
|------|------|------|
| 请求不存在 | `request_id` 错误 | 返回 `1002` |
| 请求已完成 | 仍允许查询 | 返回最终状态与最终桩号 |
| 请求已取消 | 允许查询 | `status = CANCELLED` |
| 请求已过号 | 允许查询 | `status = NO_SHOW` |

## 6.4 确认到场

### 请求

`POST /api/request/confirm_arrival`

```json
{
  "request_id": "REQ001",
  "confirm_time": "2026-03-30T10:09:00"
}
```

### 请求字段说明

| 字段 | 类型 | 必填 | 含义 |
|------|------|------|------|
| `request_id` | string | 是 | 请求编号 |
| `confirm_time` | string | 是 | 用户确认到场时间 |

### 响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "request_id": "REQ001",
    "status": "CONFIRMED",
    "station_id": "FAST_01"
  }
}
```

### 可能情况与处理

| 情况 | 说明 | 处理 |
|------|------|------|
| 请求未被叫号 | 非法确认 | 返回 `1003` |
| 已过号 | 超时未确认 | 返回 `1004` |
| 已取消 | 请求已取消 | 返回 `1005` |
| 桩故障 | 刚确认时桩故障 | 返回 `1006`，触发故障重排 |

## 6.5 取消排队

### 请求

`POST /api/request/cancel_queue`

```json
{
  "request_id": "REQ001",
  "cancel_time": "2026-03-30T10:05:00"
}
```

### 可能情况与处理

| 情况 | 说明 | 处理 |
|------|------|------|
| 当前状态为 `WAITING` | 正常取消 | 状态变更为 `CANCELLED` |
| 当前状态为 `CALLED` | 已叫号但未充电 | 允许取消，释放预留桩 |
| 当前状态为 `CHARGING` | 已开始充电 | 返回 `1003`，应调用中断充电接口 |
| 当前状态已完成 | 无法取消 | 返回 `1003` |

## 6.10 批量模拟

### 请求

`POST /api/batch/simulate`

```json
{
  "test_case_id": "BATCH_001",
  "scenario": {
    "fast_station_count": 2,
    "slow_station_count": 3,
    "waiting_area_capacity": 6,
    "station_queue_mode": "UNIFORM_CAPACITY",
    "station_queue_capacity": 3
  },
  "users": [
    {
      "user_id": "U001",
      "request_time": "2026-03-30T10:00:00",
      "charge_mode": "FAST",
      "request_energy": 20.0,
      "cancel_queue": false,
      "cancel_time": null,
      "confirm_arrival_delay_seconds": 60,
      "interrupt_charge": false,
      "interrupt_time": null,
      "leave_delay_seconds": 120
    }
  ]
}
```

### 请求字段说明

| 字段 | 类型 | 含义 |
|------|------|------|
| `test_case_id` | string | 测试用例编号 |
| `scenario` | object | 本次验收场景参数 |
| `scenario.fast_station_count` | integer | 快充桩数量 |
| `scenario.slow_station_count` | integer | 慢充桩数量 |
| `scenario.waiting_area_capacity` | integer | 等待区车位总容量，由快充池和慢充池共享 |
| `scenario.station_queue_mode` | string | `UNIFORM_CAPACITY` 或 `STATION_SNAPSHOT` |
| `scenario.station_queue_capacity` | integer/null | 每个桩的统一排队容量 |
| `scenario.station_snapshots` | array/null | 每个桩的初始状态快照 |
| `users` | array | 批量用户集合 |
| `user_id` | string | 用户编号 |
| `request_time` | string | 提交时间 |
| `charge_mode` | string | 充电模式 |
| `request_energy` | number | 申请充电量 |
| `cancel_queue` | boolean | 是否取消排队 |
| `cancel_time` | string/null | 取消时间 |
| `confirm_arrival_delay_seconds` | integer | 收到叫号后延迟多少秒确认到场 |
| `interrupt_charge` | boolean | 是否中断充电 |
| `interrupt_time` | string/null | 中断时间 |
| `leave_delay_seconds` | integer | 充电结束后延迟多少秒挪车 |

补充规则：

- `scenario.waiting_area_capacity` 为系统全局等候区容量
- 快充等待池和慢充等待池共享该容量，而不是分别独立计数
- `users[*].request_time` 是批量模拟事件排序和回放的唯一时间基准

### 响应

```json
{
  "test_case_id": "BATCH_001",
  "summary": {
    "total_users": 2,
    "completed_users": 2,
    "avg_wait_seconds": 320,
    "avg_finish_seconds": 1850,
    "total_finish_seconds": 3700,
    "station_utilization": 0.81
  },
  "results": [
    {
      "user_id": "U001",
      "detail": {},
      "bill": {}
    }
  ]
}
```

### 响应字段说明

| 字段 | 含义 |
|------|------|
| `summary.total_users` | 用户总数 |
| `summary.completed_users` | 完成流程用户数 |
| `summary.avg_wait_seconds` | 平均等待秒数 |
| `summary.avg_finish_seconds` | 平均完成秒数 |
| `summary.total_finish_seconds` | 所有用户完成时间总和 |
| `summary.station_utilization` | 充电桩利用率 |
| `results` | 每个用户的详单与账单结果 |

### 可能情况与处理

| 情况 | 说明 | 处理 |
|------|------|------|
| JSON 结构不完整 | 缺少必填字段 | 返回 `1001` |
| 同一 `user_id` 重复 | 测试用例非法 | 返回 `1001` |
| 时间字段非法 | 无法排序回放 | 返回 `1001` |
| `station_queue_mode` 非法 | 不属于支持的两种模式 | 返回 `1001` |
| `station_snapshots` 与桩数量冲突 | 快照与场景参数不一致 | 返回 `1001` |
| 并发冲突 | 同时刻多个事件 | 按统一优先级回放处理 |

## 八、并发与批量模拟处理约定

### 8.1 批量模拟回放原则

服务端必须按以下方式处理批量模拟：

1. 先根据 `scenario` 初始化桩数量、等待区容量和队列解释方式
2. 若输入为桩级快照，则先转换为内部初始状态
3. 再按 `request_time` 排序
4. 同一时刻事件按固定优先级执行
5. 每处理一个事件，就更新一次系统状态
6. 保证同一个桩不会被分配给两个用户

### 8.2 同时刻事件优先级

建议固定为：

1. 故障事件
2. 取消事件
3. 过号判定
4. 到场确认
5. 开始充电
6. 结束充电
7. 挪车确认
8. 新请求进入

# 智能充电桩调度计费系统：调度模块输入输出约定

> 版本：V3.0
> 状态：最终冻结
> 冻结日期：2026-04-19
> 维护人：组长A
> 目标：统一调度模块对内输入、输出、触发点与模式边界

## 一、调度模块职责

调度模块负责：

- 普通模式下的等候区到桩队列分配
- 故障发生后的重调度
- 故障恢复后的重调度
- 扩展模式下的联合分配或整批分配

调度模块不负责：

- 注册登录
- 页面展示
- HTTPS、Nginx 等部署细节

## 二、输入结构

### 2.1 场景输入

```json
{
  "fast_station_count": 3,
  "slow_station_count": 2,
  "waiting_area_capacity": 6,
  "charging_queue_len": 2,
  "dispatch_mode": "NORMAL",
  "fault_dispatch_mode": "TIME_ORDER"
}
```

### 2.2 普通模式调度输入

至少包含：

- 当前等候区同模式队列
- 当前各同模式充电桩队列
- 当前各同模式桩的剩余服务时间
- 待调度车辆的请求电量

### 2.3 故障调度输入

至少包含：

- 故障桩编号
- 故障桩模式
- 当前故障策略
- 故障桩未完成队列
- 其它同类型桩中未开始充电的车辆
- 当前等候区状态

### 2.4 扩展模式输入

`EXT_SINGLE_BATCH` 额外需要：

- 当前所有空位集合
- 同模式候选车辆集合

`EXT_FULL_BATCH` 额外需要：

- 当前整批车辆集合
- 全部桩能力集合

## 三、输出结构

### 3.1 普通模式输出

```json
{
  "dispatch_mode": "NORMAL",
  "scheduled_request_id": "REQ0001",
  "scheduled_queue_number": "F3",
  "target_station_code": "FAST_02",
  "target_queue_position": 2,
  "estimated_wait_seconds": 900,
  "estimated_start_time": "2026-04-19T10:15:00",
  "estimated_finish_time": "2026-04-19T10:55:00"
}
```

> `estimated_wait_seconds / estimated_start_time / estimated_finish_time` 与 [冻结接口文档.md](冻结接口文档.md) §5.4 的用户可见字段一一对应；调度模块不得只保留内部计算值而无法外推。

### 3.2 故障调度输出

至少包含：

- `fault_dispatch_mode`
- `paused_waiting_area_calling`
- `rescheduled_requests`
- `target_station_codes`
- `target_queue_positions`

### 3.3 扩展模式输出

至少包含：

- `dispatch_mode`
- `batch_id`
- `scheduled_requests`
- `station_assignments`
- `total_wait_seconds`
- `total_charge_seconds`

## 四、触发时机

### 4.1 普通模式

在以下事件后触发：

1. 新请求创建
2. 某桩队列出现空位
3. 等候区请求取消
4. 等候区请求修改模式
5. 等候区请求修改电量
6. 某车辆完成充电
7. 某车辆提前结束充电

### 4.2 故障模式

在以下事件后触发：

1. 某桩故障
2. 故障恢复
3. 故障重调度过程中某可用桩再出现空位

### 4.3 扩展模式

`EXT_SINGLE_BATCH`：

- 同时出现多个空位时触发

`EXT_FULL_BATCH`：

- 到达车辆数等于总车位数时触发整批调度

> 本触发点表与 [功能设计.md](功能设计.md) §7.4 一致；如出现差异以功能设计为最终用户可见口径。

## 五、硬性约束

1. 普通模式下必须先取等候区头车，再决定进入哪个同模式桩队列
2. 普通模式下不能跨快慢充调度
3. `ChargingQueueLen` 包含充电位
4. 故障发生后必须暂停等候区叫号
5. 故障恢复固定按时间顺序重排
6. 扩展模式与普通模式互斥
7. 扩展模式下不考虑请求修改与故障

## 六、与前端/接口层的对应要求

调度模块输出必须能支撑以下公开字段：

- `queue_number`
- `front_waiting_count`
- `station_code`
- `station_queue_position`
- `estimated_wait_seconds`
- `estimated_start_time`
- `estimated_finish_time`

因此后续实现中不允许只保留内部结构而无法映射回这些用户可见字段。

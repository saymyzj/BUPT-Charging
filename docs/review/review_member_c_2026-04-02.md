### 1. 依赖配置有硬错误

表现：

- `frontend/package.json` 中 `vue-router` 版本不对

结论：

- 先修这个，不然工程都不一定能跑起来

### 2. 文档夸大了当前完成度

表现：

- `frontend/README.md` 写“车主端全流程接口联调已完成”
- `frontend/冻结接口文档_前端实现版.md` 写“7 个核心接口全联调完成”

结论：

- 当前不能这样表述
- 应改成：
  - 页面已实现
  - 请求层已封装
  - 部分接口已开始对接
  - 管理端待接后端

### 3. 文档路径和真实路由不一致

表现：

- 文档写 `/user/request`、`/user/status`
- 实际工程是 `/user/workspace`、`/user/task`

结论：

- 必须按真实代码改文档

### 4. 文档写的请求拦截器逻辑和代码不一致

表现：

- 文档写“返回 data.data 并统一弹错误提示”
- 实际 `request.js` 只是返回 `response.data`

结论：

- 文档和代码必须统一

### 5. 当前口径仍然是旧的“中期验收 / 联调基准”

表现：

- `frontend/README.md`
- `frontend/冻结接口文档_前端实现版.md`

都还在使用旧口径。

结论：

- 必须统一改成当前“最终验收 / 系统验收”口径

### 6. 根路由行为和文档说明还有冲突

表现：

- 实际 `/` 直接跳 `/admin/overview`
- 文档更强调以 `/login` 为统一入口

结论：

- 这不是当前最大问题
- 但必须在下一轮统一

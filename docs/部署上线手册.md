## 一、部署拓扑

```
Browser
  |
  | HTTPS
  v
clouDNS 域名 --> 阿里云 ECS
                  |
                  | :443
                  v
                 Nginx
                  |
                  +--> 前端静态文件 (/var/www/charging/frontend)
                  +--> /api 反代 --> Flask (127.0.0.1:5000)
                                       |
                                       v
                                    SQLite (/var/lib/charging/charging_system.db)
```

- 单台 ECS
- 单域名同源
- HTTPS 强制
- 不引入 OSS

## 二、ECS 初始化步骤

1. 购买阿里云 ECS（中国香港/海外区域），1 vCPU / 2GB 即可
2. 安装操作系统：Ubuntu 22.04 LTS
3. 开放安全组：仅放行 22 / 80 / 443
4. 登录 ECS，执行系统更新：
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```
5. 创建服务账户：
   ```bash
   sudo useradd -r -m -d /var/lib/charging -s /bin/bash charging
   ```
6. 创建部署目录：
   ```bash
   sudo mkdir -p /var/www/charging/frontend
   sudo mkdir -p /opt/charging/backend
   sudo mkdir -p /var/lib/charging/backup
   sudo chown -R charging:charging /var/lib/charging /opt/charging
   sudo chown -R www-data:www-data /var/www/charging
   ```

## 四、后端 systemd service 模板

文件路径：`/etc/systemd/system/charging-backend.service`

```ini
[Unit]
Description=Charging Pile Backend (Flask)
After=network.target

[Service]
Type=simple
User=charging
Group=charging
WorkingDirectory=/opt/charging/backend
EnvironmentFile=/opt/charging/backend/.env
ExecStart=/opt/charging/backend/venv/bin/python -m app
Restart=on-failure
RestartSec=5
StandardOutput=append:/var/log/charging/backend.stdout.log
StandardError=append:/var/log/charging/backend.stderr.log

[Install]
WantedBy=multi-user.target
```

配置文件 `/opt/charging/backend/.env`（启动参数从这里加载）：

```env
FAST_CHARGING_PILE_NUM=3
TRICKLE_CHARGING_PILE_NUM=2
WAITING_AREA_SIZE=6
CHARGING_QUEUE_LEN=2
DISPATCH_MODE=NORMAL
FAULT_DISPATCH_MODE=TIME_ORDER
DB_PATH=/var/lib/charging/charging_system.db
LISTEN_HOST=127.0.0.1
LISTEN_PORT=5000
LOG_DIR=/var/log/charging
```

创建日志目录并启用：

```bash
sudo mkdir -p /var/log/charging
sudo chown charging:charging /var/log/charging
sudo systemctl daemon-reload
sudo systemctl enable charging-backend
sudo systemctl start charging-backend
```

## 五、Nginx 反代配置模板

文件路径：`/etc/nginx/sites-available/charging.conf`

```nginx
server {
    listen 80;
    server_name charging.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name charging.example.com;

    ssl_certificate     /etc/letsencrypt/live/charging.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/charging.example.com/privkey.pem;

    root /var/www/charging/frontend;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }
}
```

启用并加载：

```bash
sudo ln -sf /etc/nginx/sites-available/charging.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 六、clouDNS 域名解析步骤

1. 登录 clouDNS，在已持有的域名下创建子域：`charging.example.com`
2. 记录类型 `A`，值填 ECS 公网 IP
3. TTL 保持默认 300
4. 等待解析生效：`dig charging.example.com +short`

证书申请：

```bash
sudo certbot --nginx -d charging.example.com --non-interactive --agree-tos -m ops@example.com
```

`certbot` 会自动写入 SSL 配置并重载 nginx。

## 七、SQLite 备份脚本与定时任务

备份脚本 `/opt/charging/scripts/backup_db.sh`：

```bash
#!/usr/bin/env bash
set -euo pipefail
TS=$(date +%Y%m%d_%H%M%S)
DB=/var/lib/charging/charging_system.db
DST=/var/lib/charging/backup/charging_${TS}.db
sqlite3 "$DB" ".backup '$DST'"
find /var/lib/charging/backup -name 'charging_*.db' -mtime +14 -delete
```

设置权限并接入 cron：

```bash
sudo chmod +x /opt/charging/scripts/backup_db.sh
sudo crontab -u charging -e
# 追加一行：每天凌晨 3 点备份
0 3 * * * /opt/charging/scripts/backup_db.sh
```

发布前手工额外备份一次：

```bash
sudo -u charging /opt/charging/scripts/backup_db.sh
```

## 八、发布流程（手动）

1. 本地或开发机执行前端构建，产出 `dist/`
2. 上传前端：
   ```bash
   rsync -az --delete dist/ charging@<ECS>:/var/www/charging/frontend/
   ```
3. 上传后端：
   ```bash
   rsync -az --delete backend/ charging@<ECS>:/opt/charging/backend/app/
   ```
4. 远程刷新依赖（如有变更）：
   ```bash
   ssh charging@<ECS> '/opt/charging/backend/venv/bin/pip install -r /opt/charging/backend/app/requirements.txt'
   ```
5. 执行一次数据库备份（见七）
6. 重启服务：
   ```bash
   sudo systemctl restart charging-backend
   sudo systemctl reload nginx
   ```

## 九、回滚预案

四条独立回滚链路：

### 9.1 前端回滚

保留上一版目录 `frontend.prev`，回滚：

```bash
sudo rm -rf /var/www/charging/frontend
sudo mv /var/www/charging/frontend.prev /var/www/charging/frontend
sudo systemctl reload nginx
```

### 9.2 Nginx 配置回滚

```bash
sudo cp /etc/nginx/sites-available/charging.conf.prev /etc/nginx/sites-available/charging.conf
sudo nginx -t
sudo systemctl reload nginx
```

### 9.3 Flask 后端回滚

```bash
sudo rm -rf /opt/charging/backend/app
sudo mv /opt/charging/backend/app.prev /opt/charging/backend/app
sudo systemctl restart charging-backend
```

### 9.4 SQLite 回滚

```bash
sudo systemctl stop charging-backend
sudo -u charging cp /var/lib/charging/backup/charging_<timestamp>.db /var/lib/charging/charging_system.db
sudo systemctl start charging-backend
```

> 四条链路互相独立，任何一条失败都不会阻塞另外三条。

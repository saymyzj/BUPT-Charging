## 一、本轮改动的核心目标

本轮改动不是继续堆功能，而是先把以下两件事做实：

1. 严格对齐冻结接口文档、调度模块输入输出约定、系统验收规范中的公共结构
2. 把当前对成员B、成员C暴露的接口和数据结构稳定下来，后续算法继续增强时不再反复改公共边界

也就是说：

**从这一版开始，算法内核允许继续增强，但公共接口不应再频繁变动。**

## 二、本轮新增的稳定边界

### 2.1 新增统一结构构造器

新增文件：

- `backend/app/services/contract_builders.py`

该模块统一构造以下冻结结构：

1. 请求状态查询响应
2. 详单 `detail`
3. 账单 `bill`
4. 批量模拟 `summary`

意义：

- 后续不再允许在不同路由、不同服务里手写不一致的返回结构
- 成员B不需要再猜算法模块返回什么字段
- 成员C不需要再跟着后端零散字段变化反复改页面

### 2.2 当前稳定的调度入口

以下函数继续视为当前阶段稳定入口：

1. `predict_request_by_id`
2. `reschedule_waiting_requests`
3. `trigger_reschedule_for_event`
4. `simulate_batch_case`
5. `advance_request_state_for_status_view`

说明：

- 成员A后续可以继续改内部评分公式、选桩策略、重算策略
- 但不应随意改这些入口的函数名、核心参数语义和返回核心字段

## 三、本轮已做的关键收口

### 3.1 `GET /api/request/status/{request_id}` 回到冻结字段

当前状态查询响应固定为：

1. `request_id`
2. `status`
3. `station_id`
4. `estimated_wait_seconds`
5. `last_called_time`

这次收口的意义是：

- 不再继续暴露 04-02 联调阶段为页面临时补出来的一批扩展字段
- 成员C后续页面如果还依赖这些临时字段，应尽快改回冻结字段口径
- 之后若要新增字段，必须先改冻结文档，再改代码

### 3.2 `confirm_arrival` 恢复冻结语义

当前：

- `POST /api/request/confirm_arrival` 成功后返回 `status = CONFIRMED`
- 不再直接对外返回 `CHARGING`

内部处理改为：

1. 用户确认到场后，请求进入 `CONFIRMED`
2. 充电桩进入 `RESERVED`
3. 到达预计开始时间后，再由统一状态推进逻辑进入 `CHARGING`

这样做的意义是：

- 对外语义重新和冻结接口文档一致
- 对内仍保留完整闭环，不需要新增一个新的“开始充电接口”

### 3.3 状态推进从“路由打补丁”改成“统一同步逻辑”

当前统一状态推进逻辑支持：

1. `WAITING -> CALLED`
2. `CALLED -> NO_SHOW`
3. `CONFIRMED -> CHARGING`
4. `CHARGING -> COMPLETED`

并在关键推进点触发相应重调度：

1. 叫号后重排剩余等待请求
2. 过号后重调度
3. 开始充电后重调度
4. 充电结束后重调度

意义：

- 不再把“状态推进”和“接口返回拼接”混在一起
- 后续成员A继续完善状态机时，主要修改服务层，不应逼着成员B/C改公共接口

### 3.4 批量模拟输出结构已对齐冻结文档

当前 `POST /api/batch/simulate` 已收口为：

1. `summary` 只保留冻结要求字段
2. `results[*].detail` 使用冻结详单结构
3. `results[*].bill` 使用冻结账单结构

本轮同时去掉了旧的非冻结输出口径，例如：

- 自定义 `failed_users`
- 旧版批量详单中的 `submit_time` / `status` / `actual_start_time` 等旧字段结构

意义：

- 成员B后续继续完善模拟逻辑时，不应再改 `summary/detail/bill` 外形
- 成员C后续如果做批量模拟展示，应直接按当前结构接，不要再使用旧版字段名

### 3.5 批量模拟输入校验补强

本轮补充了以下校验：

1. `request_time` 必须是 ISO 8601 时间格式
2. `user_id` 不允许重复
3. `STATION_SNAPSHOT` 模式下 `station_snapshots` 数量必须与桩数一致

意义：

- 尽量把与验收规范明确冲突的输入提前挡在接口外

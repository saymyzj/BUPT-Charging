## 2. 当前已实现接口

### 2.1 已可真实调用

| 接口 | 方法 | 路径 | 当前状态 |
|------|------|------|----------|
| 健康检查 | GET | `/health` | 已实现 |
| 提交充电请求 | POST | `/api/request/create` | 已实现 |
| 查询请求状态 | GET | `/api/request/status/{request_id}` | 已实现 |
| 取消排队 | POST | `/api/request/cancel_queue` | 已实现 |
| 确认到场 | POST | `/api/request/confirm_arrival` | 已实现 |
| 中断充电 | POST | `/api/request/interrupt_charge` | 已实现 |
| 确认挪车 | POST | `/api/request/confirm_leave` | 已实现 |
| 获取详单账单 | GET | `/api/request/result/{request_id}` | 已实现 |
| 支付确认 | POST | `/api/request/pay` | 已实现 |
| 充电桩总览 | GET | `/api/stations/overview` | 已实现 |
| 批量模拟 | POST | `/api/batch/simulate` | 已实现 |

### 2.2 当前已对齐的关键口径

- `request_time` 使用客户端传入业务时间
- `waiting_area_capacity` 为全局共享等候区容量
- 批量模拟按正式 `scenario + users` 结构接收
- `GET /health` 使用裸结构返回
- 批量模拟返回 `summary + results`

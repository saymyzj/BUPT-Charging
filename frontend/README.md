### 1.3 登录与权限控制特别说明（前端模拟）

**注意：由于目前后端（成员B）尚未实现真实的用户登录、注册及权限鉴权接口，前端目前采用了纯静态模拟的跳转逻辑，以便于UI测试和主体业务流程的预览。**

- **如何模拟登录跳转**：
  - **管理员登录**：在登录页（`/login`）的用户名输入框中输入 `admin`，密码任意（不可为空），点击登录即可进入**管理员控制台**（`/admin/overview`）。
  - **普通车主登录**：在登录页的用户名输入框中输入**除 `admin` 以外的任何字符**（如 `user`、`test`），密码任意（不可为空），点击登录即可进入**普通车主工作台**（`/user/workspace`）。
- **无强制拦截预览**：目前前端并未在全局路由守卫中强行绑定“必须登录后才能访问”（无 Token 强制拦截）。你也可以直接在浏览器地址栏输入相应的绝对路由（如 `/user/task` 或 `/admin/records`）来自由跳转和预览各功能页面。
- **无强制拦截预览（但基路径 /user 与 /admin 会引导到登录）**：目前后端未实现鉴权，前端为方便 UI 测试保留了自由预览能力。但为避免误导用户直接访问管理或用户入口，路由配置做了如下约定：
   - 直接访问 `http://<host>/admin` 或 `http://<host>/user` 会被重定向到 `http://<host>/login`（即访问基路径会先展示登录页）。
   - 若你需要直接预览某个具体页面，可以使用完整子路径访问，例如 `/admin/overview`、`/admin/records`、`/user/workspace`、`/user/task` 等，这些子路由仍然可以直接打开用于页面展示和联调（但不会进行真实鉴权检查）。
   - 正式对接后端鉴权后，会把登录态（Token）写入本地并通过路由守卫限制未登录访问。

## 2. 前端通用规范

### 2.1 基础网络配置

**核心依赖：** Vue 3 + Vite + Element Plus + Axios + Vue Router
**Base URL 代理：**
```javascript
// vite.config.js 中已配置，解决跨域
'/api': {
  target: 'http://localhost:8080',
  changeOrigin: true
}
```

### 2.2 统一响应拦截处理

前端 src/api/request.js 已全局封装拦截器，严格要求服务端遵循以下返回：

**成功处理：**
```json
{
  "code": 0,
  "message": "success",
  "data": {  }
}
```

**异常处理机制：**
当前 request.js 的响应拦截器只会直接返回 response.data，并将 HTTP 异常（网络错误/状态码异常）通过 Promise.reject 抛出。具体的业务 code 异常（如 code !== 0）由调用 API 的业务组件或单独接口封装包通过校验处理，全局拦截器本身并未强制拦截和弹窗。

### 2.3 枚举映射定义（已冻结）

前端 src/constants/enums.js 已对齐服务端枚举：

**CHARGE_MODE（充电模式）：**
- FAST - 快充
- SLOW - 慢充

**STATION_STATUS（图表与看板展示状态）：**
- IDLE / RESERVED / CHARGING / WAITING_TO_LEAVE / FAULT

## 5. 本地运行与联调验证指南

为了验证当前前端与服务端的对接成果，请按照以下步骤启动项目并进行测试：

### 5.1 启动服务端 (成员B / 组长A)
前端的 `/user` 车主端流程强依赖于后端的 API (运行在 `8080` 端口)。
1. 打开终端，进入服务端目录：
   ```bash
   cd BUPT-Charging/BUPT-Charging-feature-b-backend-integration/backend
   ```
2. 安装依赖（如已安装可跳过）：
   ```bash
   pip install -r requirements.txt
   ```
3. 启动 `Flask` 调度后端：
   ```bash
   python run.py
   ```
   *服务将运行在 `http://127.0.0.1:8080`。*

### 5.2 启动前端 (成员C)
1. 打开新终端，进入前端目录：
   ```bash
   cd frontend
   ```
2. 安装依赖（首次运行）：
   ```bash
   npm install
   ```
3. 启动 `Vite` 开发服务器：
   ```bash
   npm run dev
   ```
   *控制台会输出本地访问地址，通常为 `http://localhost:3000` 或 `3002`。*

### 5.3 验证访问流程
- **验证车主端 (已对接真实API)**：
  1. 在浏览器打开 `http://localhost:<你的前端端口>/login`。
   2. 输入任意用户名（除 `admin`）及密码点击登录，即可进入车主工作台 `/user/workspace`。也可以直接访问 `/user/task` 来预览任务页面。
  3. 切换快充/慢充，选择需求电量并提交请求。
  4. 提交后会自动跳转至当前任务 `/user/task`，观察时间线状态轮询、剩余时间倒计时的自动化推演效果。
  5. 进行状态流转操作：尝试点击**取消排队**或**确认离场**。前端 Axios 会自动往后端 `8080` 发放真实 API 操作来更新真实状态。

 - **查看管理端 (静态UI展示)**：
   1. 直接访问基路径 `http://localhost:<你的前端端口>/admin` 会被重定向到 `http://localhost:<你的前端端口>/login`（这是路由配置上的保护性约定，避免误操作）。如需直接预览管理端页面，请打开具体子路径，例如：
      - `http://localhost:<你的前端端口>/admin/overview`
      - `http://localhost:<你的前端端口>/admin/records`
   2. 验证导航栏、监控看板、充电桩状态图表以及动态记录分页是否渲染正常。（注意：目前管理端数据为纯前端 Mock 演示，下一步等待后端 `/api/admin` 接口解冻即可直接对接）。

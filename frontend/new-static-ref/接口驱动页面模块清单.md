> 基准文档：`docs/冻结接口文档.md`、`docs/openapi.yaml`、`docs/功能设计.md`、`docs/增量设计.md`、`docs/系统验收接口与测试输入规范说明.md`、`docs/故障与扩展场景指南.md`、`docs/小组分工指南.md`。

## 2. 用户端模块

### 2.1 登录注册 F-U01

接口：

- `POST /api/auth/register`
- `POST /api/auth/login`

必须包含：

- 登录：`username`、`password`
- 注册：`username`、`password`、`battery_capacity`
- 根据返回 `role` 区分进入用户端或管理端

不需要：

- 营销式复杂登录表单
- 余额、支付账户、注册审批

### 2.2 提交充电请求 F-U02

接口：

- `POST /api/request/create`

字段：

- `request_time`
- `charge_mode`: `FAST` / `SLOW`
- `request_energy`: `(0, 300]` kWh

状态约束：

- 同一用户同一时间最多一个未终态请求。
- 若已有 `WAITING_AREA / QUEUED / CHARGING`，提交入口应禁用或提示当前有活跃请求。

UI 模块：

- 充电模式选择
- 请求电量输入
- 当前活跃请求提示
- 错误码提示：`1004` 等候区满、`1005` 当前模式无可用桩、`1008` 电量非法

### 2.3 查看请求状态 F-U03

接口：

- `GET /api/request/status/{request_id}`

必须展示字段：

- `queue_number`
- `charge_mode`
- `request_status`
- `front_waiting_count`
- `station_code`
- `station_queue_position`
- `estimated_wait_seconds`
- `estimated_start_time`
- `estimated_finish_time`

状态展示应严格按 7 个冻结状态：

- `WAITING_AREA`
- `QUEUED`
- `CHARGING`
- `COMPLETED`
- `COMPLETED_EARLY`
- `CANCELLED`
- `FAULT_INTERRUPTED`

### 2.4 修改请求 F-U04 / F-U05

接口：

- `PUT /api/request/mode`
- `PUT /api/request/energy`

只允许状态：

- `WAITING_AREA`

UI 规则：

- `WAITING_AREA`：展示“修改充电模式”“修改请求电量”
- `QUEUED / CHARGING`：隐藏或禁用修改入口，并提示只能提前结束后重新提交
- 修改模式后排队号会重新生成并进入新模式队尾
- 修改电量后排队号保持不变

### 2.5 取消 / 提前结束 F-U06 / F-U07

接口：

- `POST /api/request/cancel`
- `POST /api/request/stop`

状态规则：

- `WAITING_AREA`：使用 `cancel`，终态 `CANCELLED`，不生成详单
- `QUEUED / CHARGING`：使用 `stop`，终态 `COMPLETED_EARLY`，生成详单

UI 必须避免：

- 把充电区取消显示成 `CANCELLED`
- 把等候区取消和充电区提前结束混成同一个文案

### 2.6 详单 F-U08

接口：

- `GET /api/request/detail/{request_id}`

字段：

- `detail_id`
- `detail_generated_at`
- `station_code`
- `actual_energy`
- `charge_duration_seconds`
- `start_time`
- `stop_time`
- `charge_fee`
- `service_fee`
- `total_fee`
- `request_status`

UI 模块：

- 订单基础信息
- 充电桩与时间段
- 实际电量与时长
- 电费、服务费、总费用
- `COMPLETED / COMPLETED_EARLY / FAULT_INTERRUPTED` 状态说明

## 3. 管理端模块

### 3.1 系统配置 F-A04 / F-A05 / F-J01

接口：

- `GET /api/admin/system/config`
- `PUT /api/admin/system/dispatch-mode`
- `PUT /api/admin/system/fault-dispatch-mode`

必须展示：

- `fast_station_count`
- `slow_station_count`
- `waiting_area_capacity`
- `charging_queue_len`
- `dispatch_mode`
- `fault_dispatch_mode`

交互规则：

- 四个基础参数只读，不允许运行时修改。
- `dispatch_mode` 可切换：`NORMAL / EXT_SINGLE_BATCH / EXT_FULL_BATCH`
- `fault_dispatch_mode` 可切换：`PRIORITY / TIME_ORDER`

### 3.2 充电桩状态 F-A01 / F-A02 / F-A06

接口：

- `GET /api/admin/stations`
- `POST /api/admin/stations/{station_code}/start`
- `POST /api/admin/stations/{station_code}/shutdown`
- `POST /api/admin/stations/{station_code}/fault`
- `POST /api/admin/stations/{station_code}/recover`

必须展示字段：

- `station_code`
- `charge_mode`
- `station_status`: `RUNNING / SHUTDOWN / FAULT`
- `current_request_id`
- `queue_length`
- `total_charge_count`
- `total_charge_seconds`
- `total_charge_energy`

按钮规则：

- `SHUTDOWN`：可启动
- `RUNNING` 且空闲、队列为空：可关闭
- `RUNNING`：可标记故障
- `FAULT`：可故障恢复
- 关闭失败错误码 `1007` 要有明确提示：该桩未处于可关闭状态

### 3.3 单桩队列 F-A03

接口：

- `GET /api/admin/stations/{station_code}/queue`

必须展示：

- `user_id`
- `battery_capacity`
- `request_energy`
- `queue_number`
- `queue_wait_seconds`

UI 重点：

- 这是管理员查看固定桩队列的核心页面，不是人工分配页面。
- 队列包含当前 `CHARGING` 和 `QUEUED` 请求。

### 3.4 用户与车辆维护 F-A07

接口：

- `GET /api/admin/users`
- `GET /api/admin/users/{user_id}`
- `PUT /api/admin/users/{user_id}/battery-capacity`

列表字段：

- `user_id`
- `username`
- `battery_capacity`
- `role`
- `created_at`
- `has_active_request`

交互规则：

- 仅允许无活跃请求用户修改车辆电池容量。
- 若用户有活跃请求，禁用编辑并提示错误码 `1010` 语义。
- 单用户详情需要展示 `historical_details` 历史详单摘要。

### 3.5 报表 F-A08

接口：

- `GET /api/admin/reports?granularity=day|week|month`

支持粒度：

- `day`
- `week`
- `month`

字段：

- `time_key`
- `station_code`
- `total_charge_count`
- `total_charge_seconds`
- `total_charge_energy`
- `total_charge_fee`
- `total_service_fee`
- `total_fee`

### 3.6 批量模拟 / 验收回放 F-J02 / F-J03 / F-J04

接口：

- `POST /api/test/batch-simulate`
- `GET /api/health`

页面模块：

- 场景参数表单：`fast_station_count`、`slow_station_count`、`waiting_area_capacity`、`charging_queue_len`、`dispatch_mode`、`fault_dispatch_mode`
- 用户输入列表：`user_id`、`request_time`、`charge_mode`、`request_energy`
- 事件输入：`FAULT`、`RECOVER` 等故障/恢复场景事件
- 结果区：最终状态、桩队列、详单、错误码

## 4. 当前静态页差距

| 当前文件 | 状态 | 问题 | 下一步 |
|---|---|---|---|
| `landing-b.html` | 可保留 | 登录注册入口符合 F-U01，但登录后跳转还未接静态页面 | 登录成功示意跳转用户/管理端 |
| `user-dashboard.html` | 部分符合 | 已有状态、请求、订单，但请求字段用了 SOC，不符合接口 `request_energy` | 改成 `charge_mode + request_energy`，补修改模式/电量/取消/提前结束 |
| `queue-status.html` | 部分符合 | 展示了队列，但还没有 `WAITING_AREA` 修改动作与 `QUEUED/CHARGING` 差异 | 按状态机拆操作区 |
| `order-detail.html` | 部分符合 | 结构接近详单，但需要字段完全对齐 §9 | 改成详单字段字典版 |
| `admin-dashboard.html` | 需要重构 | 当前仍是“异常中心”思路，缺系统配置、单桩队列、用户维护、报表、模式切换 | 改为接口驱动的管理端工作台 |

## 5. 推荐实现顺序

1. 修正 `user-dashboard.html`：字段对齐 `POST /api/request/create` 和 `GET /api/request/status/{request_id}`。
2. 修正 `queue-status.html`：按 `WAITING_AREA / QUEUED / CHARGING` 展示可用动作。
3. 修正 `order-detail.html`：按详单字段字典展示，不展示旧账单字段。
4. 重构 `admin-dashboard.html`：以系统配置、桩状态、单桩队列入口、报表摘要为主。
5. 新增 `admin-config.html`、`admin-stations.html`、`admin-users.html`、`admin-reports.html`、`admin-batch-simulate.html`。
6. 所有页面统一错误码提示，不出现文档外接口或旧接口。

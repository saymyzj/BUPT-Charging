## 3. 与旧版 V2.0 接口差异

V2.0 使用的以下旧接口已**全部替换**，不再使用：

| 旧接口 (V2) | 替换为 (V3) |
|---|---|
| `POST /api/request/cancel_queue` | `POST /api/request/cancel` |
| `POST /api/request/confirm_arrival` | 已移除（V3 调度模型无叫号确认） |
| `POST /api/request/interrupt_charge` | `POST /api/request/stop` |
| `POST /api/request/confirm_leave` | 已移除（V3 完成即结算） |
| `GET /api/request/result/{id}` | `GET /api/request/detail/{id}` |
| `POST /api/request/pay` | 已移除（V3 详单即最终账单） |

## 4. 前端通信约定

### 4.1 Axios 配置

- 请求层：`src/api/request.js`（base URL + 拦截器 + Bearer token）
- 接口层：`src/api/charging.js`（22 个 API 函数）
- 成功时返回 `response.data`
- HTTP 异常通过 `Promise.reject` 抛出
- 业务层检查 `data.code !== 0` 处理错误

### 4.2 鉴权

- 登录成功后 `localStorage` 存储：`auth_token`, `user_role`, `user_id`, `username`
- 路由守卫检查 token + role，未登录跳 `/login`，角色不匹配自动重定向
- 登出清除所有 localStorage 项

### 4.3 状态枚举

- 统一使用 `src/constants/enums.js`（V3 §9 状态字典）
- 请求状态：`WAITING_AREA`, `QUEUED`, `CHARGING`, `COMPLETED`, `COMPLETED_EARLY`, `CANCELLED`, `FAULT_INTERRUPTED`
- 充电桩状态：`RUNNING`, `SHUTDOWN`, `FAULT`
- 调度模式：`NORMAL`, `EXT_SINGLE_BATCH`, `EXT_FULL_BATCH`
- 故障策略：`PRIORITY`, `TIME_ORDER`

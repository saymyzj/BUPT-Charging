## 3. 接口详细对接说明

### 3.1 提交充电请求
- **接口**：`POST /api/request/create`
- **前端触发**：`/user/request` -> 点击「提交请求」
- **组装参数**：从表单中获取 `charge_mode` (单选) 与 `request_energy` (输入框)，`request_time` 由前端生成 ISO 8601 格式时间传入。

### 3.2 查询请求状态（核心轮询）
- **接口**：`GET /api/request/status/{request_id}`
- **前端触发**：进入 `/user/status` 时启动定时器进行轮询。
- **UI更新机制**：将返回的 `status` 喂给组件层，匹配进度环/进度条以及对应的按钮显隐状态。

### 3.3 取消排队
- **接口**：`POST /api/request/cancel_queue`
- **前端触发**：当前状态为 `PENDING` / `WAITING` / `CALLED` 时，显示的「取消排队」按钮。

### 3.4 确认到场、中断充电与确认挪车
- **前端触发**：分别对应状态机流转期间出现的交互按钮。传参由前端自动提取当前 `request_id` 及通过 JS 生成 ISO 8601 当前时间戳执行 POST 请求。
